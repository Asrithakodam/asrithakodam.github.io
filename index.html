Key Features
🕹️
One-Tap Physics

Single-input gameplay with Rigidbody gravity and upward impulse on tap. Tight, responsive feel with adjustable gravity scale for tuning difficulty.

Rigidbody AddForce FixedUpdate
🔧
Pipe Spawner

Pipes spawn at randomised vertical positions at fixed intervals. Object pooling recycles off-screen pipes, keeping the runtime memory footprint minimal.

Object Pooling Random Range Coroutines
🏆
Score & High Score

Score increments each time the bird passes a pipe gap. Best score persists across sessions via PlayerPrefs with a live on-screen display.

PlayerPrefs OnTriggerEnter2D UI Text
💀
Collision & Game Over

BoxCollider2D on the bird detects pipe and ground hits instantly. A Game Over screen appears with current score and a restart option – no scene reload required.

05
Design Decisions
WHY?
Random Gap Height vs Fixed

Fixed gaps become pattern-learnable after one run. Random heights keep each attempt genuinely different.

WHY?
Single-Input Control

One-tap/spacebar control ensures zero barrier to entry. Players instantly understand the mechanic but spend hours mastering it.

WHY?
No Lives, Instant Restart

Frustration vanishes with one-click restart. The low friction to retry is key to the "one more run" addiction loop.

WHY?
Object Pooling for Pipes

Instantiating/destroying GameObjects per pipe causes garbage collection spikes. Pooling recycles them for smooth performance.
